# 国际化 (i18n)

YanClaw 前端内置轻量 i18n 系统，无外部依赖。当前支持中文 (`zh`) 和英文 (`en`)。

## 架构

```
packages/web/src/i18n/
├── index.tsx              # I18nProvider + useI18n() hook
└── locales/
    ├── zh.json            # 中文翻译
    └── en.json            # 英文翻译
```

### 核心 API

| 导出 | 类型 | 说明 |
|---|---|---|
| `I18nProvider` | React Component | 包裹 App 根组件，提供 i18n Context |
| `useI18n()` | Hook → `{ locale, setLocale, t }` | 获取当前语言、切换语言、翻译函数 |
| `Locale` | `"zh" \| "en"` | 语言标识类型 |

### 语言检测优先级

1. `localStorage` 中的 `yanclaw_locale`
2. 浏览器语言 (`navigator.language`)
3. 默认 `en`

## 翻译文件结构

翻译文件使用 JSON，支持嵌套 key，通过 `.` 分隔访问：

```json
{
  "common": {
    "save": "保存",
    "cancel": "取消"
  },
  "nav": {
    "chat": "聊天"
  }
}
```

组件中使用：`t("common.save")` → `"保存"`

### Key 命名规范

| 前缀 | 用途 | 示例 |
|---|---|---|
| `common.*` | 通用按钮/标签 | `common.save`, `common.cancel` |
| `nav.*` | 侧边栏导航 | `nav.chat`, `nav.settings` |
| `theme.*` | 主题切换 | `theme.light`, `theme.dark` |
| `language.*` | 语言切换 | `language.label`, `language.zh` |
| `onboarding.*` | 引导页面 | `onboarding.welcome.title` |
| `providers.*` | 模型供应商 | `providers.anthropic.name` |
| `页面名.*` | 各页面专属文案 | `agents.title`, `channels.status.online` |

## 新增语言

### 步骤

1. 复制 `locales/en.json` 为 `locales/<code>.json`（如 `ja.json`）
2. 翻译所有 value，**不要修改 key**
3. 在 `index.tsx` 中注册：

```tsx
import ja from "./locales/ja.json";

export type Locale = "zh" | "en" | "ja";

const messages: Record<Locale, Record<string, unknown>> = { zh, en, ja };
```

4. 在 `detectLocale()` 中添加检测逻辑：

```tsx
if (browserLang.startsWith("ja")) return "ja";
```

5. 在 `language-toggle.tsx` 的 `LOCALES` 数组中添加选项：

```tsx
const LOCALES: { value: Locale; label: string }[] = [
  { value: "zh", label: "中文" },
  { value: "en", label: "EN" },
  { value: "ja", label: "日本語" },
];
```

6. 如果超过 2 种语言，建议将 `language-toggle.tsx` 的 cycle 逻辑改为下拉选择。

### 验证清单

- [ ] 所有 key 在新语言文件中都有对应翻译
- [ ] `bun run build` 无报错
- [ ] 切换语言后页面文案正确刷新

## 新增翻译 Key

在组件中添加新文案时：

1. 在 **所有** locale 文件中添加对应 key 和翻译
2. 组件中使用 `const { t } = useI18n()`，调用 `t("section.key")`
3. 如果 key 不存在，`t()` 会返回 key 本身作为 fallback

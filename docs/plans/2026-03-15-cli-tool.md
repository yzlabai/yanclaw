## C1. 子命令路由框架

### 问题

现有 `cli.ts` 使用 `process.argv[2]` 做单层命令分发，不支持子命令（如 `yanclaw agents list`）和选项解析（如 `--json`、`-i`）。

### 方案

手写两层路由 + 选项解析工具函数，零外部依赖。

```typescript
// 命令签名
type Command = {
  desc: string;
  usage?: string;
  run: (args: string[], flags: Flags) => Promise<void> | void;
};

// 两层路由
const commands: Record<string, Command | Record<string, Command>> = {
  serve:    { desc: "...", run: cmdServe },
  chat:     { desc: "...", run: cmdChat },
  agents:   { list: ..., show: ..., create: ..., delete: ... },
  channels: { list: ..., connect: ..., disconnect: ... },
  // ...
};
```

**选项解析**：

```typescript
// parseFlags(["--json", "-i", "--agent", "main", "hello world"])
// → { flags: { json: true, i: true, agent: "main" }, positional: ["hello world"] }
```

**全局选项**：

| 选项 | 说明 |
|------|------|
| `--json` | 输出 JSON 格式（便于管道处理） |
| `--api <url>` | 覆盖 API 地址 |
| `--help`, `-h` | 显示帮助 |

## C2. `chat` — 终端对话

### 问题

当前 CLI 无法与 Agent 对话，必须打开浏览器。

### 方案

调用 `POST /api/chat/send`（SSE 流式），终端逐 token 渲染。

**命令格式**：

```bash
# 单次对话
yanclaw chat "帮我总结昨天的工作"
yanclaw chat --agent code "重构这个函数"

# 交互式 REPL
yanclaw chat -i
yanclaw chat -i --agent code --session my-project

# 管道模式（stdin）
cat error.log | yanclaw chat "分析这个错误"
echo "hello" | yanclaw chat --agent translator
```

**选项**：

| 选项 | 缩写 | 说明 | 默认值 |
|------|------|------|--------|
| `--interactive` | `-i` | 交互式 REPL 模式 | false |
| `--agent` | `-a` | 目标 Agent ID | `main` |
| `--session` | `-s` | 会话 Key | `agent:{agentId}:cli` |
| `--preference` | `-p` | 模型偏好 (fast/quality/cheap) | `default` |
| `--no-tools` | | 不显示工具调用详情 | false |

**SSE 事件渲染**：

| 事件 | 终端输出 |
|------|----------|
| `delta` | 直接写入 stdout（逐 token） |
| `thinking` | `[thinking] ...`（灰色，可选隐藏） |
| `tool_call` | `⚙ tool_name(args)` |
| `tool_result` | `  → result`（截断到 200 字符） |
| `done` | 换行 + `[tokens: {prompt}→{completion}]` |
| `error` | 红色 `Error: message` |
| `recall` | `📎 recalled N memories` |

**交互模式**：

```
YanClaw CLI — Agent: main — Session: agent:main:cli
Type /help for commands, /exit to quit.

> 帮我查看 package.json
⚙ file_read("package.json")
  → {"name": "yanclaw", ...}

这个项目使用了以下依赖...
[tokens: 1234→567]

> /exit
```

**REPL 命令**：

| 命令 | 说明 |
|------|------|
| `/exit`, `/quit` | 退出 |
| `/clear` | 清除当前会话 |
| `/agent <id>` | 切换 Agent |
| `/session <key>` | 切换会话 |
| `/cancel` | 取消当前运行 |

**管道检测**：`!process.stdin.isTTY` → 读取全部 stdin 追加到消息末尾。

## C3. `agents` — Agent 管理

```bash
yanclaw agents list                          # 列出所有 Agent
yanclaw agents show main                     # 查看详情
yanclaw agents create --id coder --name "编程助手" --model claude-sonnet-4-20250514
yanclaw agents delete coder                  # 删除
```

---

## C4. `channels` — 通道管理

```bash
yanclaw channels list                        # 通道列表 + 状态
yanclaw channels connect telegram bot_prod   # 连接
yanclaw channels disconnect telegram bot_prod
```

---

## C5. `sessions` — 会话管理

```bash
yanclaw sessions list                        # 列出会话
yanclaw sessions list --agent main --limit 10
yanclaw sessions show <key>                  # 查看消息历史
yanclaw sessions export <key>                # 导出 JSON
yanclaw sessions export <key> --format md    # 导出 Markdown
yanclaw sessions delete <key>
```

---

## C6. `config` — 配置管理

```bash
yanclaw config show                          # 显示当前配置
yanclaw config get gateway.port              # 读取字段
yanclaw config set gateway.port 8080         # 修改字段
yanclaw config edit                          # 打开 $EDITOR 编辑
```

---

## C7. `cron` — 定时任务

```bash
yanclaw cron list                            # 列出任务
yanclaw cron run daily-summary               # 立即执行
```

yanclaw memory stats                         # 统计
